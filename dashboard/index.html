How It Works
plain english
01
Data Collection
Prometheus scrapes every pod every 15 seconds. We keep a 30-minute rolling window — ~120 data points per pod — enough to detect memory leaks and CPU trends before they cause failures.
02
Feature Engineering
12 features are computed: memory slope (is it growing?), acceleration (growing faster?), CPU throttle ratio, network error rate, restart rate, and an anomaly score from IsolationForest.
03
Prediction & Action
GradientBoosting outputs a 0–1 risk score. Above 0.85 triggers immediate action, 0.75 triggers remediation, 0.50 triggers watch. Operator scales, restarts, or cordons automatically.
